Design a web application called “Capsular”.

  

IMPORTANT:

  

This project is currently in product discovery and UX design phase.

  

Do NOT focus on backend implementation, authentication logic, API integration, database design, Supabase, Firebase, Stripe, or any production-ready functionality.

  

This is NOT a coding task.

  

This is NOT a full-stack application task.

  

Focus exclusively on:

  

- product design

- UX

- information architecture

- user flows

- screen layouts

- interaction design

- wireframes

- visual hierarchy

  

The output should feel like an advanced clickable wireframe or UX prototype.

  

---

  

PRODUCT OVERVIEW

  

Capsular is a Personal Wardrobe Operating System designed for people who own a capsule wardrobe and want to reduce clothing-related decision fatigue.

  

The goal is NOT fashion discovery.

  

The goal is NOT social interaction.

  

The goal is NOT shopping.

  

The goal is helping users decide what to wear in less than 30 seconds.

  

Target users:

  

- Minimalists

- Young professionals

- Capsule wardrobe enthusiasts

- Productivity-focused individuals

  

Core philosophy:

  

Less decisions.

Better style.

  

---

  

MAIN USER PROBLEMS

  

1. I don't know what to wear tomorrow.

  

2. I don't know which clothes are currently available.

  

3. I don't know which clothes are in laundry.

  

4. I don't know which clothes I wear most often.

  

5. I don't know what is missing from my wardrobe.

  

6. I want to plan my outfits in advance.

  

---

  

PRODUCT PRINCIPLES

  

1. Reduce decisions

  

2. Simplicity over complexity

  

3. Quality over quantity

  

4. Planning over spontaneity

  

5. Practicality over fashion trends

  

6. AI is an assistant, not the product

  

---

  

GENERATE THE FOLLOWING SCREENS

  

1. Dashboard

  

Goal:

  

Help users understand today's and tomorrow's outfit instantly.

  

Include:

  

- Today's outfit

- Tomorrow's outfit

- Outfit readiness

- Weather

- Clothes requiring washing

- Quick actions

- Weekly overview

  

Avoid:

  

- charts

- analytics dashboards

- KPIs

  

---

  

2. Wardrobe

  

Goal:

  

Help users understand what is available.

  

Include:

  

- Search

- Filters

- Categories

- Colors

- Statuses

  

Display:

  

- Clothing image

- Name

- Category

- Color

- Availability

- Times worn

- Last worn

  

Statuses:

  

- Available

- Laundry

- Unavailable

  

---

  

3. Outfit Builder

  

Goal:

  

Guide users through creating a complete outfit.

  

Use predefined slots:

  

- Top

- Bottom

- Shoes

- Outerwear (optional)

  

Allow:

  

- multiple layers

- multiple clothing pieces per category

  

Display:

  

- favorites

- metadata

- availability status

- times worn

  

Indicate:

  

- missing outfit parts

- outfit completeness

  

---

  

4. Calendar Planner

  

Goal:

  

Allow users to plan clothing decisions ahead of time.

  

Prioritize:

  

- month view

- weekly planning

  

Display:

  

- outfit name

- outfit status

- outfit category

  

Clicking a day should reveal:

  

- outfit details

- outfit readiness

- assigned clothing items

  

Avoid:

  

- oversized clothing thumbnails

  

---

  

5. Saved Outfits

  

Goal:

  

Quick reuse of favorite outfit combinations.

  

Include:

  

- favorites

- categories

- outfit tags

- readiness state

  

---

  

6. Profile

  

Goal:

  

Prepare for future personalization and recommendation features.

  

Account Information:

  

- First Name

- Last Name

- Email

  

Personal Information:

  

- Date of Birth

- Country

- City

  

Body Profile:

  

- Height

- Weight

- Body Type

  

Preferences:

  

- Preferred Styles

- Favorite Colors

- Typical Occasions

  

Shopping Preferences:

  

- Shopping Frequency

- Price Sensitivity

  

Wardrobe Goals:

  

- Reduce Decisions

- Build Capsule Wardrobe

- Shop Smarter

- Track Usage

- Plan Outfits

  

Capsule Wardrobe Status:

  

- Just Starting

- Transitioning

- Mostly Capsule

- Fully Capsule

  

All non-account fields should be optional.

  

---

  

7. Onboarding

  

Design a lightweight onboarding flow.

  

Allow users to select:

  

- goals

- style preferences

- occasions

- wardrobe maturity

  

The experience should resemble modern onboarding flows used in premium consumer applications.

  

Users should be able to skip any step.

  

---

  

VISUAL STYLE

  

- Apple-inspired

- Premium

- Minimalistic

- Calm

- Neutral colors

- Generous whitespace

- Elegant typography

- Clean card layouts

- Modern SaaS aesthetic

  

Avoid:

  

- social media look

- Pinterest look

- marketplace look

- fashion influencer aesthetic

- e-commerce appearance

---

  

FINAL GOAL

  

Design the best possible UX for helping users decide what to wear, plan outfits, and manage a capsule wardrobe.

  

Focus on usability and information hierarchy rather than implementation details.
