Do not redesign Capsular from scratch.

  

The current design direction is already approved.

  

Your task is to refine and optimize the UX of the following areas only:

  

1. Dashboard

2. Outfit Builder

3. Calendar

4. Saved Outfits

  

Think like a senior product designer working on a production-ready MVP.

  

The goal of Capsular is:

  

"Help users decide what to wear with minimal effort."

  

Do not introduce unnecessary features.

  

Do not redesign navigation.

  

Do not introduce social features, shopping, marketplace functionality, influencers, feeds or fashion trends.

  

Focus entirely on decision reduction and wardrobe management.

  

---

  

DASHBOARD

  

Current observations:

  

- Today's outfit should be the primary focus.

- Tomorrow's outfit remains important but secondary.

- System Alerts are valuable.

- Laundry reminders are valuable.

- Users should understand what to wear within 5 seconds.

  

Improve:

  

- visual hierarchy

- action clarity

- outfit readiness visibility

- decision support

  

Explore:

  

- Today vs Tomorrow layout

- Status indicators

- Quick actions

- Weekly planning shortcuts

  

---

  

OUTFIT BUILDER

  

This is the highest priority screen.

  

Current open questions:

  

- Should users be allowed multiple tops?

- Should users be allowed multiple outerwear pieces?

- Should layering be a dedicated concept?

- How should outfit completeness be presented?

  

Design the most intuitive approach.

  

Requirements:

  

- Required slots:

  - Top

  - Bottom

  - Shoes

  

- Optional:

  - Outerwear

  - Layers

  

Support realistic outfits such as:

  

T-Shirt

+ Overshirt

+ Coat

  

Display:

  

- Times worn

- Availability

- Favorites

- Suggested pieces

  

Make outfit creation feel effortless.

  

---

  

CALENDAR

  

Current direction:

  

Monthly planning appears more useful than weekly planning.

  

Refine the calendar experience.

  

Requirements:

  

- Month view as primary layout

- Day details panel

- Outfit readiness indicators

- Laundry conflicts

- Weather awareness

  

Explore:

  

- Month / Week toggle

- Drag and drop planning

- Quick outfit assignment

  

The calendar should feel like an outfit planning tool rather than a traditional calendar.

  

---

  

SAVED OUTFITS

  

Treat saved outfits as reusable decision shortcuts.

  

Users should think:

  

"I have a meeting tomorrow."

  

instead of:

  

"I need to build another outfit."

  

Display:

  

- Outfit categories

- Occasion tags

- Favorites

- Readiness status

- Usage frequency

  

Explore ways to make saved outfits the fastest path toward daily outfit decisions.

  

---

  

PROFILE

  

Keep the current profile structure.

  

Do not redesign it significantly.

  

It is already aligned with future personalization features.

  

---

  

DESIGN PRINCIPLES

  

Capsular is:

  

- calm

- minimalist

- practical

- premium

  

Capsular is NOT:

  

- social media

- Pinterest

- e-commerce

- fashion marketplace

  

The application should feel like a Personal Wardrobe Operating System.

  

Generate the best possible MVP UX based on the existing design direction, not a new concept.
